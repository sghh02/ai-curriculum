# 第4章: Git / 作業を安全に残す

## 1. この章でやること
この章では、Gitを難しく学びません。
まずは「安心して試せるようにする道具」として使います。

AIと一緒にコードを書くと、変更が増えやすくなります。
そのときに「何が変わったか」「いつ戻れるか」が分かるようにするのがGitです。

## 2. まず覚えること
最初は次の5つで十分です。

- `git status`
- `git diff`
- `git add`
- `git commit`
- `git log --oneline`

この5つが使えれば、最初の学習には十分役立ちます。

## 3. Gitをどう使うか
毎回、次の流れで進めます。

1. 作業前に `git status` を見る
2. 変更したあとに `git diff` を見る
3. 問題なければ `git add` する
4. まとまりごとに `git commit` する
5. 最後に `git log --oneline` で履歴を見る

## 4. なぜ必要か
Gitがあると、次のことがしやすくなります。

- AIが加えた変更を確認できる
- どこで壊れたか追いやすい
- 少しずつ進めた記録が残る

## 5. この段階では深くやらないこと
この章では、難しい操作は扱いません。

- 複雑なブランチ運用
- rebaseの詳細
- 履歴のきれいな整理

今必要なのは、怖がらずに記録を残すことです。

## 6. AIの使い方
GitについてもAIに聞いて構いません。

質問の例:

- 「この変更は1回でコミットしてよい？」
- 「この変更内容に合うコミットメッセージを考えて」
- 「この差分で気をつける点を教えて」

## 7. この章のゴール
- [ ] 変更前に `git status` を見る
- [ ] 変更後に `git diff` を見る
- [ ] 小さいまとまりで `git commit` する
- [ ] 履歴を見返せる

**【ワーク】**
1つ小さな変更をして、`status -> diff -> add -> commit -> log` を1回やってみてください。

---
**【この章のまとめ】**
- Gitは難しいものとしてではなく、安心して試すための道具として使います。
- AIと作るときほど、差分確認と小さな記録が大事です。
- 最初は5つのコマンドだけで十分です。
