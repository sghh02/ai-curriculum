# 第2章: 始める前に整理する

## 1. この章でやること
第1章で決めたテーマを、そのまま始めると途中で迷いやすくなります。
この章では、始める前に最低限の整理をします。

ここでいう整理は、難しい設計書を書くことではありません。
「何をやるか」「何は今やらないか」「どこまでできたら今回はOKか」を決めることです。

## 2. まず決めること
最初の1歩を始める前に、次の5つを決めます。

1. 誰のため、または何のためか
2. どんな場面で役立つか
3. できたと言える状態
4. 今回は作らないもの
5. まだ分からないこと

## 3. 難しく考えすぎない
初心者はここで止まりやすいですが、完璧に決める必要はありません。
今必要なのは、最初の1回分の開発に足りるだけの整理です。

たとえば、次のように書ければ十分です。

```text
誰のため、または何のためか:
- 自分の学習のため

どんな場面で役立つか:
- 今日やることを忘れそうなとき

できたと言える状態:
- やることを3つ入れて一覧で見られる

今回は作らないもの:
- 締切通知
- あとで増やしたい追加要素

まだ分からないこと:
- データの保存方法
```

## 4. 「できたと言える状態」を先に決める
ここがとても大事です。
初心者が途中で苦しくなるのは、「どこまで作れば終わりか」が決まっていないことが多いからです。

`できたと言える状態` は、あとで自分で確認できる言葉で書きます。

良い例:

- 入力した内容が一覧に出る
- ページを開き直しても内容が残る
- 空のまま送信したらエラーが出る

弱い例:

- 使いやすくする
- 便利にする
- いい感じに仕上げる

## 5. AIの使い方
この段階のAIには、整理した内容の抜けを見つけてもらいます。

質問の例:

- 「この整理で足りない点を初心者向けに指摘して」
- 「このテーマで最初はやらないほうがいいことを教えて」
- 「できたと言える状態を、もっと分かりやすい言葉に直して」
- 「このままだと途中で広がりすぎそうな部分はどれ？」

## 6. この章のゴール
この章の終わりでは、次がそろっている状態を目指します。

- テーマの説明
- 始める前の整理メモ
- できたと言える状態
- 今回は作らないもの
- 分からないことのメモ

## 7. 次に進む前のチェック
- [ ] 誰のため、または何のためにやるか言える
- [ ] 最初は何だけやるか言える
- [ ] 今回作らないものが決まっている
- [ ] できたと言える状態を自分で確認できる
- [ ] 分からないことを残せている

**【ワーク】**
付録の「始める前の整理メモ」を埋めてください。
そのあとAIに「初心者が最初に作るなら広すぎる部分を削って」と依頼し、1回小さくし直してください。

---
**【この章のまとめ】**
- 始める前の整理は、初心者が迷わないために必要です。
- `できたと言える状態` と `今回は作らないもの` を先に決めると進みやすくなります。
- AIは整理の抜けを見つける相手として使えます。
