# 第2章：要件整理の極意（人間の仕事）

AIが的外れな回答をする最大の理由は**「文脈（コンテキスト）の欠如」**です。
人間同士の「言わなくてもわかる前提」はAIには一切通用しません。AIに文脈をインストールするための「要件整理」こそが、人間の最も重要な仕事です。

## 1. 最強のフレームワーク：「5W1H」×「As-Is / To-Be」
時代が変わっても腐らない最強の要件整理術は、現状把握と目標設定の組み合わせです。

### ステップ1：現在地の入力（5W1Hで As-Is を整理する）
自分が置かれている状況（前提条件や制約）を、抜け漏れなくAIに伝えます。
- **Who（誰が／誰に）**: 自分の立場、ターゲットは誰か？
- **What（何を）**: どんな商品、サービス、課題を扱っているか？
- **When（いつ）**: 期限、期間、タイミングは？
- **Where（どこで）**: 業界、プラットフォーム、場所は？
- **Why（なぜ）**: その課題が生まれた背景、理由は？
- **How（どのように）**: 現在のやり方、予算、リソースの制約は？

### ステップ2：目的地の入力（To-Be を設定する）
「結局、どういう状態になっていれば最高か？」というゴール（人間の意思）を示します。
- 例：「作業時間を現在の半分にしたい」
- 例：「顧客が自らSNSでシェアしたくなるような企画を作りたい」

## 2. なぜ「How」はAIに任せるのか？
要件整理で As-Is（現状）と To-Be（理想）が明確になれば、その間にあるギャップを埋める手段（How）は、AIが無限に計算してくれます。
人間は「要件を固めること」に全集中し、最適化やアイデア出しはAIに委ねましょう。
