# 第5章: 小さく試して確かめる

## 1. この章でやること
ここでは、第4章で決めた「最初の一歩」を実際に試します。
目的は、すごい作品を作ることではありません。

目的は次の流れを最後までやることです。

`動機を確かめる -> 方向を選ぶ -> 整理する -> 進め方を決める -> 小さく試す -> 確かめる -> 振り返る`

この流れを1回やり切ることで、次の学習が自分で決められるようになります。

## 2. 題材は自分の方向性に合わせる
第3章で決めた方向性に関連した小さな題材を選びます。

方向性ごとの例:

| 方向性 | 小さな題材の例 |
|---|---|
| フロントエンド | HTMLとCSSで自己紹介ページを1枚作る |
| バックエンド | Javaで入力した数を計算して返すプログラムを書く |
| モバイル | Flutterで「ボタンを押すとテキストが変わる」アプリを作る |
| テスト/QA | 簡単な関数に対してテストケースを1つ書いて実行する |
| SE | 身近な課題を要件に分解して整理メモを書く |
| インフラ | ローカルで簡単なサーバーを1つ起動して画面を表示する |

## 3. 進め方
次の順番で進めます。

1. 第4章で決めた最初の3ステップを見直す
2. 1ステップ目をやる
3. 進んだか確かめる
4. 作業記録をメモに残す
5. 2ステップ目をやる（時間があれば）
6. 振り返る

## 4. AIに任せること
AIには次のことを頼ってよいです。

- 最初のたたき台やサンプルコード
- エラーやつまずきの原因候補
- 次の一歩の候補
- 次にやることの整理

## 5. 自分で持つこと
自分で決めるのは次です。

- 最初はどこまでやるか
- 何をやって何を今はやらないか
- 本当に思った通りに動いたか
- 次に何を学ぶか

## 6. AIの出力をそのまま使わない
AIが出したコードや回答をそのまま入れるのは、一番多い失敗パターンです。
必ず次の手順で確かめます。

### 確かめ方の3ステップ
1. **読む**: AIが出したコードや説明を、1行ずつ読む。分からない行があれば「この行は何をしている？」とAIに聞く
2. **動かす**: 実際に動かして、期待した結果になるか確認する。ならなければ、どこが違うかを特定する
3. **変えてみる**: 一部を変えてもう一度動かす。変えた結果が予想通りなら、理解できている

この3ステップが、「手触りの経験」になります。
知識として知っているだけでなく、自分で触って確かめた経験があることで、次にAIに質問するときの質が上がります。

## 7. この章で残すもの
成果物だけでなく、次の記録も残します。

- 第4章で決めた進め方（見直し後）
- 試した結果
- 作業記録メモ
- 動作確認メモ（何を確認し、結果はどうだったか）
- 振り返りメモ

## 8. よくある失敗
- 最初から大きくやりすぎる
- AIが出したものをそのまま入れる
- 動いたので確認を省く
- 何を学んだか残さない
- エラーが出たとき、内容を読まずにAIに丸投げする

## 9. この章のゴール
次の状態になれば、この章は完了です。

- 自分の方向性に関連した1つの小さな題材を最後まで試せている
- AIの出力を読み、動かし、変えて確かめた経験がある
- 途中の作業記録がメモに残っている
- 次に学ぶことを書ける

**【ワーク】**
付録の「動作確認メモ」「振り返りメモ」を使って記録しながら進めてください。
自分の方向性に関連した小さな題材を1つ試してください。
AIの出力に対して「読む→動かす→変えてみる」の3ステップを実践してください。
試したあとで、AIに「次に進むなら何を1つ学ぶべきか」と聞いて、振り返りに書き足してください。

---
**【この章のまとめ】**
- 最初の目標は、大きい作品ではなく小さい前進です。
- AIの出力は「読む→動かす→変えてみる」で確かめます。
- 試して終わりではなく、振り返って次の学習につなげます。
- この1回の経験が、自分のエンジニア像を設計するための材料になります。
