# 最終ゲート：Gitカリキュラムに進むための完了判定

## この章の目的

ここまで作った設計成果物が、実務レベルで再現可能かを判定します。  
合格したら、次の Git カリキュラムへ進みます。

## 判定対象

1. `design-spec.md`
2. `design-change-log.md`
3. 各フェーズでの質問ログ（要点）

## 完了条件（合格ライン）

次をすべて満たしたら合格です。

1. 評価3軸すべてで `3/4` 以上
2. `design-spec.md` が第三者再現可能な粒度である
3. `design-change-log.md` に仕様変更対応が2件以上ある

## 実施ステップ

1. 自己採点を実施（3軸ルーブリック）
2. AIに第三者レビューを依頼し、誤読ポイントを抽出
3. 設計書を修正し、最終版を確定
4. 「Git開始前チェック」を実施

## Git開始前チェック

- [ ] 目的と成功条件が1文で定義されている
- [ ] 処理手順と条件分岐が日本語で記述されている
- [ ] 例外時の復旧手順がある
- [ ] 仕様変更時の差し替え履歴が2件以上ある
- [ ] 次に実装する機能の優先順位が決まっている

## AIに聞く例

```text
この設計書を第三者レビューして。
「誤読される箇所」「実装時に詰まる箇所」「不足条件」を優先度順で返して。
```

```text
この設計書を渡された新メンバーが1日で作業開始できるか判定して。
不足している情報だけ箇条書きで教えて。
```
