# AIと問いから始める学び方

## 1. このプログラムは誰のためのものか
このプログラムは、プログラミングが未経験、または少し触ったことがある人のためのものです。
「何から学べばいいか分からない」「AIは使ってみたけど学習にどう使えばいいか分からない」という状態から始められるように作っています。

## 2. このプログラムのゴール
このプログラムのゴールは、コードが書ける人になることではありません。

ゴールは、**自分がどんなエンジニアになりたいかを、自分の言葉で説明できる状態**になることです。

具体的には、次の問いに自分で答えられるようになります。

- なぜ自分はエンジニアになりたいのか
- エンジニアにはどんな種類があるのか
- 自分はどの方向を目指すのか
- そのために最初に何を学べばいいのか
- 今の自分に足りないものは何か

これが、あなたの最初の「要件定義と設計」です。

## 3. 今までの学び方と、これからの学び方
今までのプログラミング学習は、`基礎 -> 応用 -> 実践` の順で進むことが多くありました。
まず文法を学び、次にフレームワークを学び、最後に作品を作る流れです。

このやり方は今でも間違いではありません。
ただ、AIがコードのたたき台を作れるようになった今、初心者にとってもっと進みやすい学び方があります。

それは、

`自分の目的を見つける -> 始める前に整理する -> AIに進め方を聞く -> 小さく試す -> 足りない基礎をその都度学ぶ`

という進め方です。

このプログラムでは、この進め方をまず **自分のキャリア** に対して実践します。

## 4. 「問いから始める学び方」とは何か
「問いから始める学び方」は、AIにうまく頼りながら、自分で考えて前に進むための土台です。

難しいことをするためのものではありません。
次のことができるようになるための考え方です。

- 自分は何をやりたいのか、なぜやりたいのかを言葉にする
- それを始める前に整理する
- AIに何を聞けば前に進めるか考える
- 小さく試して、次に何を学ぶべきか決める

このプログラムでは、この考え方を **自分のキャリアを題材にして** 身につけます。

## 5. このプログラムの進め方
このプログラムは、次の順で進みます。

1. なぜエンジニアになりたいのかを言葉にする
2. どんなエンジニアがいるのかを知る
3. 自分の方向性を整理する
4. AIと一緒に最初の一歩を決める
5. 小さく試して確かめる
6. 自分のエンジニア像を設計する

最後に出す成果物は、**自分のエンジニアとしての要件定義と設計書**です。

## 6. このプログラムで大切にする考え方
- 学習の出発点は文法ではなく、自分の目的です
- AIは答えをもらう箱ではなく、考える相手です
- 分からないことは悪いことではなく、次に学ぶことのヒントです
- 最初から大きく決めず、小さく試しながら方向を確かめます
- 正解を探すのではなく、今の自分に合った答えを見つけます

## 7. プログラム構成
このプログラムは6章＋付録の構成です。

- 第1章: `なぜエンジニアになりたいのか`
  - 自分の動機や原体験を言葉にします。
- 第2章: `どんなエンジニアがいるのか`
  - エンジニアの種類と役割を知り、選択肢を広げます。
- 第3章: `自分の方向性を整理する`
  - 自分の関心と選択肢を照らし合わせて方向を絞ります。
- 第4章: `AIと最初の一歩を決める`
  - AIに相談しながら、最初に試すことと学ぶことを決めます。
- 第5章: `小さく試して確かめる`
  - 実際に1つ小さく試し、記録をメモに残しながら確かめます。
- 第6章: `自分のエンジニア像を設計する`
  - 全体を振り返り、自分のキャリアの要件定義と設計を書きます。
- 付録: `実践ワークシート`
  - 各章で使うメモとテンプレート集です。

## 8. このプログラムで残すもの
このプログラムでは、次の記録を残します。

- なぜエンジニアになりたいかを書いたメモ
- エンジニアの種類と自分の関心の整理
- 自分の方向性の整理メモ
- AIと決めた最初の一歩
- 小さく試した記録（作業メモ含む）
- 振り返りメモ
- 自分のエンジニア像の設計書（最終成果物）

## 9. 評価の考え方
評価は、「難しいものを作れたか」ではありません。
次の点を見ます。

- 自分がなぜエンジニアになりたいか言葉にできたか
- エンジニアの選択肢を知ったうえで方向を選べたか
- AIに前に進むための質問ができたか
- 小さく試して結果を確かめられたか
- 自分のエンジニア像を具体的に書けたか

## 10. 受講者へのメッセージ
このプログラムは、いきなり専門用語を覚えるためのものではありません。
AIを使いながら、「自分はどんなエンジニアになりたいか」を自分で設計するためのプログラムです。

最初の問いは、コードの書き方ではなく「なぜエンジニアになりたいのか」です。
その問いに自分で答えられたとき、次に何を学べばよいかが見えてきます。
