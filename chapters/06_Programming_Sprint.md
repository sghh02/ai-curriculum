# 第5章: 小さく試して学ぶ

## 1. この章でやること
ここでは、今まで決めてきた内容をもとに、実際に1つ小さく試します。
目的は、すごい作品を作ることではありません。

目的は次の流れを最後までやることです。

`テーマを決める -> 始める前に整理する -> AIと進め方を決める -> 小さく試す -> 動かして確かめる -> 振り返る`

## 2. どんな題材で進めるか
最初の題材は、小さくて、自分で変化を感じやすいものが向いています。

例:

- 今日やることを3つメモできるツール
- 支出を1件ずつ記録するメモ
- よく使う文章を組み立てるツール
- 単語を少しずつ確認できるツール
- HTMLとCSSで自己紹介ページを1枚作る練習
- Pythonで1つの単純作業を自動化する練習

## 3. 進め方
次の順番で進めます。

1. 始める前の整理を見直す
2. AIに最初の進め方を相談する
3. 小さく試す
4. 進んだか確かめる
5. Gitで記録を残す
6. 振り返る

## 4. AIに任せること
AIには次のことを頼ってよいです。

- 最初のたたき台
- エラーやつまずきの原因候補
- 次の一歩の候補
- 次にやることの整理

## 5. 自分で持つこと
自分で決めるのは次です。

- 最初はどこまでやるか
- 何をやって何を今はやらないか
- 本当に思った通りに進んだか
- 次に何を学ぶか

## 6. この章で残すもの
成果物だけでなく、次の学習記録も残します。

- テーマメモ
- 始める前の整理メモ
- AIと決めた進め方
- Gitの記録
- 動作確認メモ
- 振り返りメモ

## 7. よくある失敗
- 最初から大きくやりすぎる
- AIが出したものをそのまま入れる
- 試したので確認を省く
- 何を学んだか残さない

## 8. この章のゴール
次の状態になれば、この章は完了です。

- 1つ小さな題材を最後まで試せている
- 自分で結果を確認できている
- Gitに途中の記録が残っている
- 次に学ぶことを書ける

**【ワーク】**
自分のテーマで、最初の小さい目標を1つ試してください。
試したあとで、AIに「次に進むなら何を1つ学ぶべきか」と聞いて、振り返りに書き足してください。

---
**【この章のまとめ】**
- 最初の目標は、大きい作品ではなく小さい前進です。
- AIを使っても、自分で確かめることが大事です。
- 試して終わりではなく、振り返って次の学習につなげます。
